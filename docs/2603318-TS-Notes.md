## 1. 原始类型与严格检查 (primitive.ts)

### 1.1 基础类型声明

```typescript
// 隐式类型推断
let username = "Tom";

// 显式类型声明
let teamname: string = "Barsa";
let age: number = 25;
let isTrue: boolean = false;
let d: bigint = 9007199254740991n;
```

### 1.2 何时必须显式写类型？

- **变量声明时没有初始值**
  ```typescript
  let hisName: string;
  ```

- **需要限制类型范围（联合类型）**
  ```typescript
  let id: number | string = 123;
  ```

- **类型推断不符合预期时**

### 1.3 字面量类型 (Literal Type)

字面量类型是把「具体的单个值」作为一种类型：

```typescript
// 推断为字面量类型 "Tom"
let username1: "Tom" = "Tom";
// username1 = "Jerry"; // ❌ 报错

// 显式标注为通用 string 类型
let username2: string = "Tom";
username2 = "Jerry"; // ✅ 合法
```

**注意**：`const` 声明的变量会自动推断为字面量类型：
```typescript
const username3 = "Mike"; // 类型为 "Mike"
```

### 1.4 Strict Null Check

当 `strict: true` 时，`null` 和 `undefined` 不能直接赋值给其他类型：
```typescript
let username5: string = "Mike";
// username5 = null; // ❌ 不能将类型"null"分配给类型"string"
```

## 2. 集合类型 (collections.ts)

### 2.1 数组类型

```typescript
// 两种等价写法
let numbers: number[] = [1, 2, 3];
let numbers2: Array<number> = [1, 2, 3];

// 特点：元素类型相同，长度不固定
numbers.push(99);
```

### 2.2 元组类型

```typescript
// 固定长度，固定每个位置的类型
let user: [number, string] = [1, "Tom"];

// 只读元组
let user2: readonly [number, string] = [3, "Mike"];
// user2[1] = "Jason"; // ❌ 无法为"1"赋值，因为它是只读属性
```

> **注意**：元组的"固定长度"是编译时约束，运行时可通过 `push` 改变长度。

## 3. 枚举类型 (enums.ts)

### 3.1 数字枚举

```typescript
enum Direction {
    Up,    // 0
    Down,  // 1
    Left,  // 2
    Right  // 3
}

// 双向映射
console.log(Direction.Up);      // 0
console.log(Direction[0]);      // "Up"
```

### 3.2 字符串枚举

```typescript
enum Status {
    Success = "SUCCESS",
    Error = "ERROR"
}

// ✅ 无反向映射，可读性好
```

### 3.3 常量枚举

```typescript
const enum Direction1 {
    Up, Down, Left, Right
}

// 编译后不生成 Direction1 变量，只剩数字
console.log(Direction1.Down);
```

## 4. 顶层类型与特殊类型 (topAndSpecialType.ts)

### 4.1 any - 放弃类型检查

```typescript
let value: any = "hello";
value = 123;
value = true;
console.log(value.length); // 编辑不报错，运行输出 undefined
```

### 4.2 unknown - 安全的顶级类型

```typescript
let value1: unknown = "hello";
// value1.length; // ❌ 编辑报错

// 使用断言访问
console.log((value1 as string).length);

// 或类型判断
if (typeof value1 === "string") {
    console.log(value1.length);
}
```

### 4.3 void - 没有返回值

```typescript
function logMessage(): void {
    console.log("hello");
}
```

### 4.4 never - 永远不可能

```typescript
// 情况1：抛异常
function throwError(): never {
    throw new Error("Stop");
}

// 情况2：死循环
function infiniteLoop(): never {
    while (true) {}
}

// never 用于类型收窄
function test(value: string | number) {
    if (typeof value === "string") {
        console.log("string");
    } else if (typeof value === "number") {
        console.log("number");
    } else {
        // value 类型为 never，永远执行不到
    }
}
```

## 5. 断言与字面量类型 (assertionsAndLiterals.ts)

### 5.1 断言

```typescript
let value5: unknown = "Tom";
let strLength = (value5 as string).length;
```

### 5.2 字面量类型

```typescript
let role: "TM" = "TM";
```

### 5.3 类型别名

```typescript
type Role = "admin" | "user";
let role2: Role = "admin";
```

**使用场景**：
- 多次使用同一类型（复用性）
- 修改类型时只需改一处（可维护性）
- 提高代码可读性

### 5.4 as const

```typescript
const user7 = { role: "admin" };        // role: string
const user8 = { role: "admin" as const }; // role: "admin"

const roles2 = ["admin", "user"] as const; // readonly ["admin", "user"]

// 高级用法：提取数组元素类型
type Roles = typeof roles2[number]; // "admin" | "user"
```
